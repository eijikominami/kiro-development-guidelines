# 品質保証ガイドライン

## Pre-commit

`.pre-commit-config.yaml` が存在する場合、コミット時に自動的にコード品質チェックが実行される。

## 確認の責任分担

### Kiro が実施する確認

| カテゴリ | 確認内容 |
|---------|---------|
| コード品質 | フォーマット、Lint、型チェック、テストカバレッジ |
| ドキュメント | README の手順、コード例の動作、リンクの有効性、バージョン番号 |
| 設定ファイル | 構文エラー、依存関係の整合性 |

### 人間が実施する確認

| カテゴリ | 確認内容 |
|---------|---------|
| 動作確認 | アプリケーション起動、主要機能、ユーザー体験 |
| 視覚確認 | UI 要素、レイアウト（UI がある場合） |
| ビジネス確認 | 要件充足、エッジケース、セキュリティ |

## 確認タイミング

| タイミング | 実施者 |
|-----------|--------|
| プルリクエスト作成前 | Kiro + 人間 |
| main マージ前 | Kiro + 人間 |
| タグプッシュ前 | Kiro + 人間 |

## チェックリスト

### コミット前

- [ ] コードフォーマット実行
- [ ] Lint チェック
- [ ] 基本機能テスト
- [ ] 依存関係の整合性確認
- [ ] 設定ファイルの構文チェック
- [ ] ドキュメントの整合性確認

### リリース前

- [ ] クリーン環境でのインストールテスト
- [ ] 全機能の動作確認
- [ ] アンインストールテスト
- [ ] ドキュメントの手順確認
- [ ] CI/CD の動作確認

### 緊急時対応

- [ ] 影響範囲の特定
- [ ] ロールバック手順の確認
- [ ] ユーザー通知の準備
- [ ] 修正版のテスト
